# Database Schema

This document defines the planned database schema for Limbo. Tables are grouped by build phase. Only Phase 1 and Phase 2 tables are fully specced ÔÇö later phases will be detailed when their phase begins.

---

## Phase 1 ÔÇö Auth & Identity

These tables are required for the initial auth and registration flows.

### `users`

The core user table. Uses ULIDs as the primary key (Laravel `HasUlids` trait). Replaces the default auto-increment `id`.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `username` | string | Unique handle (e.g. `voidoflimbo`) ÔÇö chosen at registration |
| `name` | string | Display name |
| `email` | string | Unique |
| `email_verified_at` | timestamp | nullable |
| `password` | string | nullable ÔÇö null for social-only accounts |
| `bio` | text | nullable ÔÇö short profile bio |
| `tier` | enum | `free`, `premium`, `loyalist` ÔÇö default: `free` |
| `profile_image` | string | nullable ÔÇö storage path |
| `profile_image_fallback` | string | nullable ÔÇö static fallback for animated formats |
| `cover_image` | string | nullable ÔÇö storage path |
| `cover_image_fallback` | string | nullable ÔÇö static fallback for animated formats |
| `two_factor_secret` | text | nullable ÔÇö already exists via migration |
| `two_factor_recovery_codes` | text | nullable ÔÇö already exists |
| `two_factor_confirmed_at` | timestamp | nullable ÔÇö already exists |
| `remember_token` | string | nullable |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

> `tier` reflects the user's current highest active entitlement. `premium` is granted on one-time server purchase; `loyalist` is granted by active recurring subscription and implicitly includes `premium` access.

---

### `social_accounts`

Stores OAuth provider credentials linked to a user. A single user may have multiple social accounts (one per provider). Used by Socialite for Google and Microsoft (Outlook) login.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `user_id` | ulid | FK Ôćĺ `users.id` (cascade delete) |
| `provider` | enum | `google`, `microsoft` (expandable to more providers later) |
| `provider_id` | string | User ID returned by the OAuth provider |
| `provider_token` | text | nullable ÔÇö OAuth access token |
| `provider_refresh_token` | text | nullable |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

> Unique constraint on `(provider, provider_id)`.

---

### `invite_codes`

Generated by server owners to grant access to their server. The Limbo server owner may also issue codes for controlled platform onboarding.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `server_id` | ulid | FK Ôćĺ `servers.id` (cascade delete) |
| `created_by` | ulid | FK Ôćĺ `users.id` ÔÇö must be the server owner |
| `code` | string | Unique, URL-safe alphanumeric code |
| `max_uses` | integer | nullable ÔÇö null means unlimited uses |
| `uses_count` | integer | default: 0 |
| `expires_at` | timestamp | nullable ÔÇö null means no expiry |
| `is_active` | boolean | default: true ÔÇö owner can deactivate |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

## Phase 2 ÔÇö Servers & Membership

Required for the Limbo server space and community features.

### `servers`

A branded community space. The `Limbo` server is seeded as the default on fresh installs.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `owner_id` | ulid | FK Ôćĺ `users.id` |
| `name` | string | Display name |
| `slug` | string | Unique, URL-safe identifier (e.g. `limbo`) |
| `description` | text | nullable |
| `logo` | string | nullable ÔÇö storage path |
| `banner` | string | nullable ÔÇö storage path |
| `is_default` | boolean | default: false ÔÇö true only for the Limbo server |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

---

### `server_members`

Tracks user membership within a server. Server owners are also recorded here with the `owner` role.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `server_id` | ulid | FK Ôćĺ `servers.id` (cascade delete) |
| `user_id` | ulid | FK Ôćĺ `users.id` (cascade delete) |
| `role` | enum | `member`, `moderator`, `owner` |
| `invite_code_id` | ulid | FK Ôćĺ `invite_codes.id`, nullable ÔÇö tracks which code was used |
| `joined_at` | timestamp | |

> Unique constraint on `(server_id, user_id)`.

---

## Phase 3 ÔÇö Social Graph

Required for follow/friend features which are referenced by the access control visibility stack.

### `follows`

Polymorphic ÔÇö a user can follow another user or a server.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `follower_id` | ulid | FK Ôćĺ `users.id` (cascade delete) |
| `followable_id` | ulid | Polymorphic target ID |
| `followable_type` | string | `App\Models\User` or `App\Models\Server` |
| `created_at` | timestamp | |

> Unique constraint on `(follower_id, followable_id, followable_type)`.

---

### `friends`

Bidirectional contact list. Not the same as mutual follows ÔÇö requires explicit mutual acceptance.

| Column | Type | Notes |
|---|---|---|
| `id` | ulid | Primary key |
| `user_id` | ulid | FK Ôćĺ `users.id` (cascade delete) |
| `friend_id` | ulid | FK Ôćĺ `users.id` (cascade delete) |
| `status` | enum | `pending`, `accepted` |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

> Unique constraint on `(user_id, friend_id)`. Application logic ensures only one direction is stored per pair (lower ID always goes in `user_id`).
